Arrays: Organizing Multiple Values

What is an array? An array is a container holding multiple values of the same type, arranged sequentially in memory. Instead of creating 100 variables (temperature1, temperature2, temperature3...), you create ONE array with 100 slots.

Real-world analogy: Think of a post office wall with 10 identical numbered mailboxes. You store letters in boxes 0-9. When you need a specific letter, you know its box number. Arrays work the same way: each "box" holds one value, accessed by its position (index).

Critical Safety Concept: Zero-Based Indexing

Arrays count from 0, not 1. An array of size 10 has valid indices 0-9. Index 10 is OUT OF BOUNDS. Accessing out-of-bounds memory corrupts other variables or crashes the program. This is one of the most dangerous parts of C—the language doesn't protect you. You're responsible for bounds checking.

Why arrays are essential in embedded systems:

  • Sensor buffers: Store 10 temperature readings, compute average to filter noise
  • LED strips: Control brightness of 50 LEDs—one array, 50 values
  • Lookup tables: Pre-calculate sin/cos/PWM values, store in array, just look up (faster than calculating)
  • Serial buffers: Receive bytes one at a time, store in array until complete message
  • Image data: Camera sends pixel data—store as 2D array (rows × columns)

Declaring and Using Arrays

Basic Declaration

uint8_t sensor_readings[10];  // Array of 10 uint8_t values
// Allocates 10 consecutive bytes of RAM
// Valid indices: sensor_readings[0] through sensor_readings[9]

Memory Layout

uint8_t buffer[5] = {10, 20, 30, 40, 50};

// Memory (consecutive addresses):
// buffer[0] = 10  (address 0x1000)
// buffer[1] = 20  (address 0x1001)
// buffer[2] = 30  (address 0x1002)
// buffer[3] = 40  (address 0x1003)
// buffer[4] = 50  (address 0x1004)

// Key: Contiguous memory!
// Accessing buffer[5] reads memory past the boundary—corruption!

Different Data Types

Memory efficiency is critical in embedded systems:

uint8_t sensor_readings[10];   // 10 bytes total (1 byte × 10)
uint16_t adc_values[256];      // 512 bytes total (2 bytes × 256)
int32_t temperature_log[100];  // 400 bytes total (4 bytes × 100)

// Microcontroller with 2KB RAM? Every byte matters!
// Choose the smallest data type that works

Initializing Arrays

Method 1: Set all values when declaring:

uint8_t command[4] = {0xAA, 0xBB, 0xCC, 0xDD};  // All values set upfront
uint16_t lookup_table[10] = {1, 4, 9, 16, 25, 36, 49, 64, 81, 100};  // Lookup table

Method 2: Declare then set individual elements:

uint8_t buffer[100];
buffer[0] = 42;
buffer[1] = 100;
buffer[99] = 255;

Method 3: Zero-initialize (all elements = 0):

uint8_t sensor_data[50] = {};  // All elements initialized to 0

Accessing Array Elements

uint8_t temps[5];

// Write values
temps[0] = 20;
temps[1] = 22;
temps[2] = 19;

// Read values
uint8_t current_temp = temps[2];  // Gets 19
printf("Temp at index 2: %d\n", temps[2]);

// Use in calculations
uint8_t average = (temps[0] + temps[1] + temps[2]) / 3;

Looping Through Arrays

uint8_t sensor_readings[10];

// Read all 10 sensors
for (uint8_t i = 0; i < 10; i++) {
    sensor_readings[i] = analogRead(SENSOR_PINS[i]);
}

// Calculate average
uint16_t sum = 0;
for (uint8_t i = 0; i < 10; i++) {
    sum += sensor_readings[i];
}
uint16_t average = sum / 10;

// Find maximum
uint8_t max = sensor_readings[0];
for (uint8_t i = 1; i < 10; i++) {
    if (sensor_readings[i] > max) {
        max = sensor_readings[i];
    }
}
printf("Max reading: %d\n", max);
⚠️ Buffer Overflow: The Classic Embedded Bug

Accessing past the array boundary corrupts memory. This is one of the oldest and most dangerous bugs in C:

uint8_t buffer[5];

// CORRECT: Valid indices
buffer[0] = 100;
buffer[4] = 200;

// DANGEROUS: Out of bounds!
buffer[5] = 50;   // Writing to memory after the array ends
               // Corrupts whatever was next in RAM

// Even worse—this often doesn't crash immediately!
// It corrupts other variables silently, causing mysterious failures later

2D Arrays: Matrices and Multi-Dimensional Data

When you need grid-like data: Image pixels (rows × columns), game boards, sensor networks arranged in a grid—use 2D arrays.

Declaring 2D Arrays

uint8_t image[240][320];      // 240 rows, 320 columns (image pixels)
uint16_t sensor_grid[8][8];  // 8×8 sensor array
int16_t matrix[3][3];        // 3×3 mathematical matrix

// Memory layout: Row by row
// Addresses are sequential:
// [0][0], [0][1], [0][2], ... [0][319],  (row 0)
// [1][0], [1][1], [1][2], ... [1][319],  (row 1)
// ... and so on

Accessing 2D Array Elements

uint8_t led_grid[4][8];  // 4 rows, 8 columns (32 LEDs arranged in grid)

// Set brightness of LED at row 1, column 3
led_grid[1][3] = 200;

// Read value
uint8_t brightness = led_grid[1][3];  // Gets 200

// Initialize grid
for (uint8_t row = 0; row < 4; row++) {
    for (uint8_t col = 0; col < 8; col++) {
        led_grid[row][col] = 0;  // All LEDs off
    }
}

Real Example: Image Processing

uint8_t image[480][640];  // 480×640 pixel image

// Read image data
for (uint16_t row = 0; row < 480; row++) {
    for (uint16_t col = 0; col < 640; col++) {
        image[row][col] = camera_read_pixel(row, col);
    }
}

// Process: Make image 50% darker
for (uint16_t row = 0; row < 480; row++) {
    for (uint16_t col = 0; col < 640; col++) {
        image[row][col] = image[row][col] / 2;  // Halve brightness
    }
}

Common Embedded Array Patterns

Pattern 1: Ring Buffer (Circular Buffer)

Use case: Fixed-size buffer that overwrites old data when full. Useful for sensor readings, audio samples, data logging.

struct RingBuffer {
    uint8_t buffer[256];
    uint16_t write_index;
    uint16_t read_index;
    uint16_t count;
};

void ring_buffer_write(RingBuffer *rb, uint8_t value) {
    rb->buffer[rb->write_index] = value;
    rb->write_index = (rb->write_index + 1) % 256;  // Wrap around
    
    if (rb->count < 256) {
        rb->count++;
    } else {
        rb->read_index = (rb->read_index + 1) % 256;  // Discard oldest
    }
}

uint8_t ring_buffer_read(RingBuffer *rb) {
    uint8_t value = rb->buffer[rb->read_index];
    rb->read_index = (rb->read_index + 1) % 256;
    rb->count--;
    return value;
}

Pattern 2: Lookup Table (LUT)

Use case: Pre-calculate complex values (sin, cos, PWM curves) and store in array. At runtime, just look up instead of computing (much faster).

// Pre-calculated sine wave (0-360 degrees, 0-255 range)
const uint8_t sin_lut[360] = {
    0, 4, 9, 13, 18, 22, 27, 31, 35, 40, ... // values from 0 to 359
};

// At runtime: instant lookup instead of expensive sin() calculation
uint8_t sine_value = sin_lut[angle];  // O(1) instead of expensive trig

// Example: PWM brightness curve (non-linear brightness looks better to eyes)
const uint8_t brightness_curve[256] = {
    0, 1, 1, 2, 2, 3, 3, 4, 4, 5, ... // Perceptual brightness mapping
};

void set_led_brightness(uint8_t perceived_brightness) {
    uint8_t pwm_value = brightness_curve[perceived_brightness];
    PWM_write(pwm_value);
}

Pattern 3: Sensor Moving Average

Use case: Store multiple sensor readings, compute average to filter noise.

struct SensorBuffer {
    uint16_t readings[10];
    uint8_t index;
};

void add_reading(SensorBuffer *buf, uint16_t new_reading) {
    buf->readings[buf->index] = new_reading;
    buf->index = (buf->index + 1) % 10;
}

uint16_t get_average(SensorBuffer *buf) {
    uint32_t sum = 0;
    for (uint8_t i = 0; i < 10; i++) {
        sum += buf->readings[i];
    }
    return sum / 10;
}

Pattern 4: State Machine Lookup

Use case: Array of function pointers for state machine transitions.

typedef void (*StateFunction)(void);

void state_idle(void);
void state_running(void);
void state_error(void);

StateFunction state_handlers[3] = {
    state_idle,
    state_running,
    state_error
};

// Execute current state handler
state_handlers[current_state]();

Key Takeaways

  • Arrays store multiple values of the same type in contiguous memory
  • Array indexing is ZERO-BASED: valid indices for size N are 0 to N-1
  • Accessing array[N] is undefined behavior—it reads/writes past the boundary
  • Use the smallest data type sufficient for your values (saves precious RAM)
  • Loops are the standard way to process array elements
  • 2D arrays organize grid-like data (images, sensor networks, game boards)
  • Common patterns: ring buffers, lookup tables, moving averages, state machines
  • Always bounds-check array access—the compiler won't catch out-of-bounds mistakes

Mini-Projects to Strengthen Your Skills

Project 1: Sensor Noise Filter

Objective: Store 8 sensor readings, calculate moving average to smooth noise.

Requirements:

  • Create array to hold 8 sensor readings
  • Read 8 sensors in a loop
  • Calculate average (sum / 8)
  • Find max and min values
  • Print results
uint16_t readings[8];
uint16_t sum = 0;
uint16_t max_reading = 0;
uint16_t min_reading = 1024;

// Read sensors
for (uint8_t i = 0; i < 8; i++) {
    readings[i] = analogRead(A0 + i);
    sum += readings[i];
    
    if (readings[i] > max_reading) max_reading = readings[i];
    if (readings[i] < min_reading) min_reading = readings[i];
}

uint16_t average = sum / 8;
printf("Avg: %d, Max: %d, Min: %d\n", average, max_reading, min_reading);

Key Learning: Use a single loop to process multiple similar items and collect statistics.

Project 2: Circular Buffer Implementation

Objective: Implement a simple ring buffer for collecting incoming sensor data.

Requirements:

  • Create fixed-size buffer (16 bytes)
  • Implement write function (wraps around when full)
  • Collect 20 sensor readings (will overwrite oldest)
  • Print last 16 readings from buffer
#define BUFFER_SIZE 16

uint8_t buffer[BUFFER_SIZE];
uint8_t write_index = 0;

void buffer_write(uint8_t value) {
    buffer[write_index] = value;
    write_index = (write_index + 1) % BUFFER_SIZE;  // Wrap around
}

// Collect 20 readings (last 16 remain in buffer)
for (uint8_t i = 0; i < 20; i++) {
    uint8_t sensor_value = analogRead(A0) >> 2;  // Convert to 0-255
    buffer_write(sensor_value);
}

// Print buffer contents
for (uint8_t i = 0; i < BUFFER_SIZE; i++) {
    printf("%d ", buffer[i]);
}
printf("\n");

Key Learning: Ring buffers use modulo operator (%) to wrap index around when reaching buffer end.

Project 3: 2D LED Grid Pattern Generator

Objective: Create and display patterns on an 8×8 LED grid using 2D array.

Requirements:

  • Create 8×8 array for LED brightness
  • Create checkerboard pattern (alternating on/off)
  • Create diagonal stripe pattern
  • Display both patterns (iterate and print)
uint8_t led_grid[8][8];

// Create checkerboard pattern
for (uint8_t row = 0; row < 8; row++) {
    for (uint8_t col = 0; col < 8; col++) {
        if ((row + col) % 2 == 0) {
            led_grid[row][col] = 255;  // ON
        } else {
            led_grid[row][col] = 0;    // OFF
        }
    }
}

// Create diagonal pattern (secondary)
for (uint8_t row = 0; row < 8; row++) {
    for (uint8_t col = 0; col < 8; col++) {
        if (row == col) {
            led_grid[row][col] = 255;  // Diagonal
        } else {
            led_grid[row][col] = 128;  // Dimmer off-diagonal
        }
    }
}

Key Learning: 2D arrays iterate with nested loops: rows first, then columns for each row.
