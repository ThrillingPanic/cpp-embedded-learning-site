Functions: Reusable Blocks of Code

What is a function? A function is a reusable block of code that performs a specific task. Instead of copying the same code 100 times, you write it once in a function and call it 100 times. Functions are the foundation of organized, maintainable code.

Why functions matter:

  • Code reuse: Write once, use many times
  • Modularity: Break complex problems into smaller, manageable pieces
  • Maintainability: Fix a bug in one place instead of searching through your entire program
  • Testability: Test functions in isolation before combining them
  • Readability: A well-named function explains what code does immediately
Function Anatomy 
return_type function_name(parameter1_type param1, parameter2_type param2) {
    // Function body: the code that does the work
    // Variables declared here are LOCAL (only exist inside function)
    
    return value;  // Return result to caller (if return_type is not void)
}

Real embedded examples where functions exist:

  • Sensor reading: `int read_temperature()` — centralizes all sensor logic in one place
  • Data processing: `void filter_noise(int *data, int size)` — processes array without duplicating logic
  • Motor control: `void set_motor_speed(uint8_t speed)` — abstracts PWM complexity
  • Serial communication: `void send_message(char *str)` — handles protocol details
  • Interrupt handlers: `void button_pressed_ISR()` — called by hardware when button pressed

Declaring and Calling Functions

Function Declaration (Prototype)

Tell the compiler about your function BEFORE main():

// Function prototypes appear at top of file
int add(int a, int b);           // Function that returns int
void print_status();              // Function that returns nothing (void)
int8_t read_sensor(uint8_t pin);  // Function with parameters and return

Function Definition

Write what the function actually does:

// Function definition
int add(int a, int b) {
    int result = a + b;
    return result;  // Return the answer to the caller
}

Calling Functions

// In main() or another function:
int sum = add(5, 3);  // Call add() with 5 and 3
printf("Sum: %d\n", sum);  // Output: Sum: 8

Example 1: Simple Function with Return Value

// Prototype
uint8_t map_percent(uint8_t value, uint8_t max_value);

// Definition
uint8_t map_percent(uint8_t value, uint8_t max_value) {
    return (value * 100) / max_value;  // Return percentage
}

// Call
uint8_t battery_percent = map_percent(current_voltage, max_voltage);
printf("Battery: %d%%\n", battery_percent);

Example 2: Function with No Return (void)

// Prototype
void blink_led(uint8_t times, uint16_t delay_ms);

// Definition
void blink_led(uint8_t times, uint16_t delay_ms) {
    for (uint8_t i = 0; i < times; i++) {
        digitalWrite(LED_PIN, HIGH);
        delay(delay_ms);
        digitalWrite(LED_PIN, LOW);
        delay(delay_ms);
    }
}

// Call
blink_led(3, 500);  // Blink 3 times, 500ms each

Example 3: Multiple Parameters

// Prototype
void configure_pwm(uint8_t pin, uint16_t frequency, uint8_t duty_cycle);

// Definition
void configure_pwm(uint8_t pin, uint16_t frequency, uint8_t duty_cycle) {
    // Set up PWM on specified pin with frequency and duty cycle
    set_pin_mode(pin, PWM);
    set_frequency(pin, frequency);
    set_duty_cycle(pin, duty_cycle);
}

// Call with multiple parameters
configure_pwm(PIN_9, 1000, 75);  // 1kHz, 75% duty cycle

Parameters: Input to Functions

Pass-by-Value: Copy the Data

When you pass a variable by value, the function gets a COPY:

void increment(int x) {
    x++;  // Increment the LOCAL copy, not the original
}

int main() {
    int count = 5;
    increment(count);
    printf("%d\n", count);  // Still 5! Function modified its own copy
}

Pass-by-Reference (Using Pointers): Modify the Original

When you need a function to modify the original variable, pass a POINTER:

void increment(int *x) {
    (*x)++;  // Dereference pointer, modify the ORIGINAL
}

int main() {
    int count = 5;
    increment(&count);  // Pass address of count
    printf("%d\n", count);  // Now 6! Function modified the original
}

Real Embedded Example: Returning Multiple Values

Problem: A function can return only ONE value. How do we get multiple outputs? Answer: Use pointers to modify multiple variables.

// Read temperature and humidity from sensor
// Can't return both, so use pointers to store both
void read_sensor_data(int16_t *temperature, uint8_t *humidity) {
    // Read from sensor/ADC
    *temperature = 25;  // Dereference and set temperature
    *humidity = 45;     // Dereference and set humidity
}

int main() {
    int16_t temp;
    uint8_t humid;
    
    read_sensor_data(&temp, &humid);  // Pass addresses
    
    printf("Temp: %d C, Humidity: %d%%\n", temp, humid);
}

Passing Arrays to Functions

Arrays are automatically passed by reference (you always get the original):

void clear_buffer(uint8_t *buffer, uint8_t size) {
    for (uint8_t i = 0; i < size; i++) {
        buffer[i] = 0;  // Modifies original array
    }
}

uint8_t my_buffer[256];
clear_buffer(my_buffer, 256);  // Clears the actual array

Scope: Where Variables Exist

Local Variables: Inside Function Only

void example_function() {
    int local_var = 100;  // Local—only exists inside this function
    // ... use local_var ...
}  // local_var ceases to exist here

// This is an ERROR: local_var doesn't exist outside the function
// printf("%d\n", local_var);  // UNDEFINED!

Global Variables: Exist Everywhere

int global_counter = 0;  // Declared at top level

void increment_counter() {
    global_counter++;  // Can access global from anywhere
}

int main() {
    increment_counter();
    printf("%d\n", global_counter);  // Can read global here too
}

Static Variables: Persist Between Calls

A static variable inside a function keeps its value between calls:

void count_calls() {
    static uint16_t call_count = 0;  // Initialized only once
    call_count++;
    printf("Function called %d times\n", call_count);
}

// In main:
count_calls();  // Output: Function called 1 times
count_calls();  // Output: Function called 2 times
count_calls();  // Output: Function called 3 times
// call_count persists between calls!
Embedded Use: Debounce Counter

Static variables are perfect for debouncing. The button debounce state persists between function calls:

uint8_t is_button_pressed() {
    static uint8_t debounce_count = 0;
    
    if (digitalRead(BUTTON_PIN) == HIGH) {
        debounce_count++;
        if (debounce_count >= 10) {
            return 1;  // Button confirmed pressed
        }
    } else {
        debounce_count = 0;
    }
    return 0;
}

Embedded-Specific Function Patterns

Pattern 1: Interrupt Service Routines (ISRs)

Functions called by hardware when events occur:

// ISR called when button pressed
void button_pressed_ISR() {
    button_flag = 1;  // Set flag
    // ISRs should be SHORT—do minimal work here
}

// Main loop checks flag
int main() {
    attachInterrupt(BUTTON_PIN, button_pressed_ISR, RISING);
    
    while (1) {
        if (button_flag) {
            handle_button_press();
            button_flag = 0;
        }
    }
}

Pattern 2: Callback Functions

Pass a function as a parameter (function pointer):

// Function pointer type
typedef void (*EventCallback)(void);

// Call a callback when event occurs
void on_timer_expired(EventCallback callback) {
    if (callback) {
        callback();  // Call the passed function
    }
}

// Define callbacks
void alarm_function() {
    printf("ALARM!\n");
}

// Register callback
on_timer_expired(alarm_function);

Pattern 3: State Machine Functions

Organize state handling with dedicated functions:

enum State { IDLE, RUNNING, ERROR };

void state_idle() {
    // Idle state logic
}

void state_running() {
    // Running state logic
}

void state_error() {
    // Error state logic
}

// State machine loop
while (1) {
    switch (current_state) {
        case IDLE: state_idle(); break;
        case RUNNING: state_running(); break;
        case ERROR: state_error(); break;
    }
}

Pattern 4: Inline Functions for Performance

Tell compiler to insert function code inline (no call overhead):

// Inline function for performance on repeated calls
inline uint8_t clamp(uint8_t value, uint8_t min_val, uint8_t max_val) {
    if (value < min_val) return min_val;
    if (value > max_val) return max_val;
    return value;
}

// When you call clamp(x, 0, 255), compiler inserts the logic directly
// No function call overhead!

Key Takeaways

  • Functions organize code into reusable, testable blocks
  • Function prototype declares it before main(); definition implements it
  • Parameters pass data INTO functions; return values send data OUT
  • Pass-by-value (default) copies data; pass-by-reference (pointers) modifies original
  • Local variables exist only inside their function; global variables exist everywhere
  • Static variables persist their value between function calls
  • ISRs (interrupt handlers) are functions called by hardware
  • Callback functions allow event-driven programming
  • Inline functions eliminate call overhead for small, frequently-used code
  • Functions are the foundation of organized embedded systems

Mini-Projects to Strengthen Your Skills

Project 1: Sensor Reading Library Functions

Objective: Create reusable functions for sensor operations.

Requirements:

  • Function to read temperature and return as integer
  • Function to read and return average of multiple samples
  • Function to check if reading is in valid range (returns 1 if valid, 0 if out of range)
  • Call all three functions and print results
int16_t read_temperature() {
    int raw = analogRead(TEMP_PIN);
    return (raw * 500) / 1024;  // Convert to celsius
}

int16_t get_average(uint8_t samples) {
    int32_t sum = 0;
    for (uint8_t i = 0; i < samples; i++) {
        sum += read_temperature();
        delay(10);
    }
    return sum / samples;
}

uint8_t is_temperature_valid(int16_t temp) {
    if (temp >= -40 && temp <= 125) {
        return 1;  // Valid range
    }
    return 0;  // Out of range
}

// In main:
int16_t temp = read_temperature();
int16_t avg = get_average(5);
if (is_temperature_valid(avg)) {
    printf("Temperature valid: %d C\n", avg);
}

Key Learning: Decompose complex tasks into reusable functions with clear purposes.

Project 2: Using Pointers to Modify Multiple Values

Objective: Write a function that returns multiple values using pointers.

Requirements:

  • Create function that takes array and calculates stats (min, max, average)
  • Use pointers to return all three statistics
  • Call function with test data and print results
void calculate_stats(uint8_t *data, uint8_t size,
                      uint8_t *min, uint8_t *max, uint8_t *avg) {
    uint16_t sum = 0;
    *min = data[0];
    *max = data[0];
    
    for (uint8_t i = 0; i < size; i++) {
        sum += data[i];
        if (data[i] < *min) *min = data[i];
        if (data[i] > *max) *max = data[i];
    }
    
    *avg = sum / size;
}

// In main:
uint8_t sensor_data[10] = {45, 50, 48, 52, 49, 51, 47, 50, 48, 49};
uint8_t minimum, maximum, average;

calculate_stats(sensor_data, 10, &minimum, &maximum, &average);

printf("Min: %d, Max: %d, Avg: %d\n", minimum, maximum, average);

Key Learning: Use pointers to return multiple values from a function.

Project 3: Persistent Counter with Static Variables

Objective: Track internal state across function calls using static variables.

Requirements:

  • Create function that counts how many times it was called
  • Use static variable to persist count
  • Call function 10 times (in loop) and verify count increments each time
uint16_t get_call_count() {
    static uint16_t count = 0;
    count++;
    return count;
}

// In main:
for (uint8_t i = 0; i < 10; i++) {
    uint16_t call_num = get_call_count();
    printf("Call #%d\n", call_num);
}

// Output:
// Call #1
// Call #2
// ... through ...
// Call #10

Key Learning: Static variables maintain state across function calls, perfect for counters and debouncing.
