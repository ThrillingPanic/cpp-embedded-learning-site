Loops in C++: The Heartbeat of Embedded Systems

What is a loop, fundamentally? A loop is a programming construct that executes the same block of code repeatedly until a specified condition becomes false. It's not magical—it's just repeated execution. But this simple concept is absolutely foundational to embedded programming and, indeed, to almost all real-world software.

The core principle: Instead of typing the same code 100 times (or 1000 times, or infinitely), you write it once and tell the computer "repeat this." This saves you from insanity and creates maintainable code. If you need to fix a bug, you fix it in one place, not in 100 copies.

Why loops are ABSOLUTELY ESSENTIAL in embedded systems: A desktop computer runs an operating system. When your program crashes, the OS detects it, shows you an error message, and maybe restarts. But a microcontroller has NO operating system. When it powers on, it runs your code and NOTHING ELSE. If your code exits or crashes, the device just sits there, dead. This means your embedded program must run continuously, forever—reading sensors, making decisions, controlling hardware—until power is cut or a reset button is pressed. This is done exclusively with loops.

The standard embedded program structure: Almost every embedded system follows this pattern:

void setup() {
    // The setup() function: called once when power is first applied
    // Configure hardware: pins, timers, communication protocols
    pinMode(LED_PIN, OUTPUT);
    pinMode(BUTTON_PIN, INPUT);
    Serial.begin(9600);  // Start serial communication
    // ... more hardware setup
}

void loop() {
    // The loop() function: called REPEATEDLY after setup() completes
    // This loop runs FOREVER until power is cut or reset is pressed
    
    // Typical cycle: Read → Process → Output → Repeat
    uint16_t sensor_value = analogRead(SENSOR_PIN);
    
    if (sensor_value > THRESHOLD) {
        digitalWrite(LED_PIN, HIGH);
        send_alert_to_server();
    } else {
        digitalWrite(LED_PIN, LOW);
    }
    
    delay(100);  // Wait 100 milliseconds, then loop back to top
    // Program now loops back and does all of this again, and again, forever
}

// Your program flow: setup() → loop() → loop() → loop() → loop() → ...
// (The loop() function executes infinitely until power is lost)

Concrete embedded examples where loops are critical:

  • Reading sensor arrays: You have 10 temperature sensors on your circuit board. Instead of writing 10 separate read statements, you loop: for(int i = 0; i < 10; i++) { int temp = readSensor(i); }
  • Communication and data reception: A serial port sends data one byte at a time. You loop to receive each byte until the complete message arrives: while(bytesReceived < expectedMessageLength) { byte = readByte(); }
  • Timing sequences: You want an LED to blink 5 times, with 500ms on and 500ms off each time. You loop: for(int i = 0; i < 5; i++) { digitalWrite(LED_PIN, HIGH); delay(500); digitalWrite(LED_PIN, LOW); delay(500); }
  • Processing stored data: You read 256 sensor samples into a buffer and need to find the average, max, and min. You loop through the buffer: for(int i = 0; i < 256; i++) { processData(buffer[i]); }
  • Waiting for events: Your device waits for a button press: while(!buttonPressed) { checkButton(); } The device will loop here—checking the button over and over—until it detects a press.
  • Data transmission: You had 50 bytes to send over UART. Instead of 50 separate send statements, you loop: for(int i = 0; i < 50; i++) { uart_send_byte(data[i]); }

The Three Types of Loops (Each Has Its Purpose)

C++ provides three different loop types. They can all do the same thing, but each is designed for a specific situation:

  • while loop: Use this when you DON'T know how many iterations you need. You repeat "until this condition changes." Example: "Keep reading sensor until it stabilizes" or "Wait until button is pressed."
  • for loop: Use this when you DO know the exact number of iterations. It's designed for counting. Example: "Loop 100 times" or "Loop through 50 array elements" or "Count from 0 to 9."
  • do-while loop: Rarely used, but critical for specific situations. The body executes at least once before checking the condition. Example: "Ask user for input, then ask again if input was invalid."

How to avoid infinite loops and common pitfalls: A loop repeats until its condition becomes false. If you make a mistake and the condition NEVER becomes false, you have an infinite loop—and your program is stuck forever. We'll see several examples of this throughout this lesson, and how to prevent it.

The while Loop: "Keep Doing This Until A Condition Changes"

How a while loop works: A while loop repeats its code block over and over as long as a specified condition is true. The moment that condition becomes false, the loop exits and the program continues with the code below the loop. This is the most fundamental loop type in programming.

Basic syntax:

while (condition) {
    // This code executes repeatedly
    // as long as condition is true
    
    // If condition is false, program jumps here:
}
// Program continues from here once loop exits

How it flows: First, check if condition is true. If yes, execute body. If no, skip to after the loop. Once the body completes, go back to the beginning and check condition again. Keep repeating until condition becomes false.

The Most Common while Loop: Counting

The simplest while loop counts from 0 to 9, executing the body 10 times. This demonstrates the critical requirement that the loop condition must eventually become false. Before the loop starts, you initialize a counter to 0. Inside the loop, you increment that counter. Each time the counter increases, the condition is re-evaluated. Once the counter reaches 10, the condition count < 10 becomes false, and the loop exits.

uint8_t count = 0;

while (count < 10) {
    Serial.print(count);
    Serial.println(" ");
    count++;
}

Serial.println("Counting complete!");

Waiting for Events (Polling)

A common embedded pattern: Wait for something to happen. This is called "polling"—constantly checking a condition until it changes. The loop checks if a button is pressed. Once the condition becomes true (button_pressed is set to 1), the loop condition !button_pressed becomes false, and the loop exits.

uint8_t button_pressed = 0;

while (!button_pressed) {
    if (digitalRead(BUTTON_PIN) == HIGH) {
        button_pressed = 1;
    }
    delay(10);
}

Serial.println("Button detected!");

Another example: Wait for sensor to stabilize. When a sensor is first powered on or activated, its readings may fluctuate. This code reads repeatedly until the same value appears 5 times in a row, indicating the sensor has stabilized.

uint16_t last_reading = 0;
uint16_t current_reading;
uint8_t stable_count = 0;

while (stable_count < 5) {
    current_reading = analogRead(SENSOR_PIN);
    
    if (current_reading == last_reading) {
        stable_count++;
    } else {
        stable_count = 0;
    }
    
    last_reading = current_reading;
    delay(50);
}

Serial.print("Stable reading: ");
Serial.println(current_reading);

Reading Data Until Complete Message

Serial communication pattern: Read bytes one at a time until you receive all bytes of a message. The special value 255 serves as an end-of-message marker. When that byte arrives, we exit the loop immediately using the break statement.

uint8_t buffer[256];
uint8_t bytes_received = 0;
uint8_t END_MARKER = 255;

while (bytes_received < 256) {
    if (serial_data_available()) {
        uint8_t byte_received = serial_read_byte();
        
        if (byte_received == END_MARKER) {
            break;
        }
        
        buffer[bytes_received] = byte_received;
        bytes_received++;
    }
}

Serial.print("Received ");
Serial.print(bytes_received);
Serial.println(" bytes");
process_message(buffer, bytes_received);

The Infinite Loop: Forever!!!!

The MOST COMMON while loop in embedded systems is the infinite loop. Your main program must run forever until power is cut. The condition 1 is always true, so the loop never exits naturally. This is the standard pattern for the main control loop of an embedded device.

while (1) {
    read_all_sensors();
    process_all_data();
    update_all_outputs();
}

More realistic embedded infinite loop: This version allows for controlled shutdown. A flag variable controls whether the loop continues, allowing the program to exit gracefully when needed.

uint8_t system_running = 1;

while (system_running) {
    read_temperature_sensor();
    read_pressure_sensor();
    read_button_inputs();
    
    update_display();
    control_heater();
    
    if (power_button_pressed_long()) {
        system_running = 0;
    }
}

power_down_hardware();
save_settings_to_eeprom();
Serial.println("System shutdown complete");

Critical Mistake: Infinite Loops You Didn't Want

⚠️ The Most Common Loop Bug: Forgetting to Update the Condition

The absolute MOST COMMON bug with while loops is that you forget to change the condition inside the loop. If the condition never becomes false, the loop never exits, and your program hangs forever.

uint8_t count = 0;
while (count < 10) {
    Serial.println("Hello!");
    count++;
}

Serial.println("Done");

How to prevent this: Before writing a while loop, ask yourself: "What must happen inside this loop to eventually make the condition false?" Then MAKE SURE that thing actually happens in your code. Common patterns:

  • Increment a counter: while (counter < MAX) → must have counter++ in loop
  • Change a flag: while (flag == 1) → must have flag = 0 in loop
  • Read until end: while (data_available) → must actually read data and set data_available = false

Another Critical Mistake: The Condition Is Always True

⚠️ Logic Error: Testing the Wrong Condition 
uint8_t data = 0;
while (data == 1) {
    Serial.println("Running!");
}

Remember: = (single equals) is assignment. == (double equals) is comparison. This is one of the most dangerous bugs in C and C++.

Exiting a while Loop Early With break

Sometimes you want to exit a loop before the condition becomes false. Use the break statement to jump immediately out of the loop, skipping any remaining code in the loop body.

uint16_t target_value = 42;
uint8_t found_index = 255;

uint8_t i = 0;
while (i < 100) {
    if (array[i] == target_value) {
        found_index = i;
        break;
    }
    i++;
}

if (found_index != 255) {
    Serial.print("Found at index: ");
    Serial.println(found_index);
}

The do-while Loop: Execute First, Check Later

The key difference from while: A normal while loop checks its condition FIRST. If the condition is false, the body never executes. A do-while loop executes the body FIRST, and only then checks the condition. This means a do-while loop ALWAYS runs at least once, even if the condition is false from the start.

Syntax:

do {
} while (condition);

Comparison with while:

while (count < 10) {
    Serial.println(count);
    count++;
}

uint8_t count = 0;
do {
    Serial.println(count);
    count++;
} while (count < 10);

Why This Matters: The Condition Starts False

Example: while loop with false condition

uint8_t count = 100;

while (count < 10) {
    Serial.println("This never prints");
    count++;
}

Serial.println("Skipped the loop");

Same code with do-while:

uint8_t count = 100;

do {
    Serial.println("This prints once!");
    count++;
} while (count < 10);

Serial.println("Loop completed");

Practical Use Case: User Input Validation

Scenario: You ask user to enter a value between 1 and 10. If they enter something invalid, ask again. You MUST ask at least once (to get input), then check if it's valid.

uint8_t user_value;
uint8_t valid_input = 0;

do {
    Serial.println("Enter a value (1-10):");
    user_value = read_from_serial();
    
    if (user_value >= 1 && user_value <= 10) {
        Serial.println("Valid!");
        valid_input = 1;
    } else {
        Serial.println("Invalid! Try again.");
    }
} while (!valid_input);

process_value(user_value);

Practical Use Case: Device Menu System

Real embedded example: Control panel menu

uint8_t menu_choice;
uint8_t user_wants_to_exit = 0;

do {
    Serial.println("\n=== DEVICE MENU ===");
    Serial.println("1: Set Temperature");
    Serial.println("2: Set Timer");
    Serial.println("3: View Status");
    Serial.println("0: Exit");
    Serial.println("Choose: ");
    
    menu_choice = read_from_serial();
    
    switch (menu_choice) {
        case 1:
            set_temperature();
            break;
        case 2:
            set_timer();
            break;
        case 3:
            view_status();
            break;
        case 0:
            user_wants_to_exit = 1;
            Serial.println("Exiting menu...");
            break;
        default:
            Serial.println("Invalid choice!");
            break;
    }
} while (!user_wants_to_exit);

Serial.println("Menu closed");

Retry Logic: Try, Then Retry If Failed

Practical scenario: Attempt to send a packet over radio. If it fails, retry up to 3 times before giving up.

uint8_t success = 0;
uint8_t retry_count = 0;
uint8_t MAX_RETRIES = 3;

do {
    Serial.print("Attempt ");
    Serial.print(retry_count + 1);
    Serial.println("...");
    
    success = send_data_packet(data);
    
    if (!success) {
        retry_count++;
        if (retry_count < MAX_RETRIES) {
            Serial.println("Failed. Retrying...");
            delay(100);
        } else {
            Serial.println("All retries exhausted.");
        }
    } else {
        Serial.println("Sent successfully!");
    }
} while (!success && retry_count < MAX_RETRIES);

if (success) {
    Serial.println("Data transmitted");
} else {
    Serial.println("Failed to send after retries");
}

When NOT to Use do-while

do-while is useful but not common. Don't use it just because the code "looks nice":

Use do-while ONLY when: You need the body to execute at least once, regardless of the condition. Examples: getting user input, retry loops, menu systems.

Use a regular while loop for: Everything else. Most loop situations don't need "execute first."

The for Loop: Counting and Iteration

When to use for: Use a for loop when you know EXACTLY how many times the body should execute. It's designed specifically for counting. The for loop is more concise than an equivalent while loop and makes your intention crystal clear: "Execute this code a specific number of times."

Basic syntax comparison:

uint8_t i = 0;
while (i < 10) {
    Serial.println(i);
    i++;
}

for (uint8_t i = 0; i < 10; i++) {
    Serial.println(i);
}

Understanding the for Loop Structure

The for loop has three parts, all in the parentheses: initialization, condition check, and increment. The initialization runs once before the loop starts. The condition is checked before each iteration. The increment runs after each iteration.

for (initialization; condition; increment) {
}

How it flows: The initialization sets up the counter. Then the loop checks if the condition is true. If so, it executes the body. After the body completes, the increment runs, and then the condition is checked again. This repeats until the condition becomes false.

for (uint8_t i = 0; i < 3; i++) {
    Serial.println(i);
}

Common for Loop Patterns

Pattern 1: Counting Forward (0 to 9)

for (uint8_t i = 0; i < 10; i++) {
    Serial.println(i);
}

Pattern 2: Counting Forward Starting at 1 (1 to 10)

for (uint8_t i = 1; i <= 10; i++) {
    Serial.println(i);
}

Pattern 3: Counting Backward (10 down to 1)

for (uint8_t i = 10; i > 0; i--) {
    Serial.println(i);
}

Pattern 4: Counting by Steps (Every Other Number)

for (uint8_t i = 0; i < 20; i += 2) {
    Serial.println(i);
}

for (uint8_t i = 0; i < 20; i += 3) {
    Serial.println(i);
}

The Most Common Use: Array Iteration

Reading through all elements in an array is the #1 use for for loops in embedded systems.

uint8_t sensor_readings[10] = {12, 45, 23, 67, 34, 89, 11, 50, 22, 99};

for (uint8_t i = 0; i < 10; i++) {
    Serial.print("Sensor ");
    Serial.print(i);
    Serial.print(": ");
    Serial.println(sensor_readings[i]);
}

Task 2: Sum All Values

uint16_t total = 0;
for (uint8_t i = 0; i < 10; i++) {
    total += sensor_readings[i];
}
uint16_t average = total / 10;
Serial.print("Average: ");
Serial.println(average);

Task 3: Find the Maximum Value

uint8_t max_value = 0;
for (uint8_t i = 0; i < 10; i++) {
    if (sensor_readings[i] > max_value) {
        max_value = sensor_readings[i];
    }
}
Serial.print("Maximum: ");
Serial.println(max_value);

Task 4: Find the Index of a Specific Value

uint8_t target = 67;
uint8_t found_index = 255;

for (uint8_t i = 0; i < 10; i++) {
    if (sensor_readings[i] == target) {
        found_index = i;
        break;
    }
}

if (found_index != 255) {
    Serial.print("Found at index: ");
    Serial.println(found_index);
} else {
    Serial.println("Not found");
}

The Infinite for Loop

You can leave any of the three parts empty to create special loops: An infinite for loop has all three parts empty, creating a loop that runs forever unless exited with a break statement. This is equivalent to while(1).

for (;;) {
    read_sensors();
    process_data();
}

For Loop With Complex Initialization and Increment

The increment part doesn't have to be i++. It can be any statement. You can even initialize and increment multiple variables in a single for loop.

for (uint8_t i = 0, j = 100; i < 10; i++, j--) {
    Serial.print(i);
    Serial.print(" ");
    Serial.println(j);
}

Critical Mistakes with for Loops

⚠️ Off-by-One Errors

Problem: Loop doesn't execute the correct number of times.

for (uint8_t i = 0; i < 10; i++) {
    array[i] = value;
}

How to prevent: If looping through 10 items, the condition should be i < 10 or i <= 9. Both work—just be consistent.

⚠️ Modifying Loop Variable Inside Body

Problem: You change i inside the loop, overriding the increment at the bottom.

for (uint8_t i = 0; i < 10; i++) {
    if (array[i] == target) {
        break;
    }
}

Nested Loops: Loop Inside a Loop

What is a nested loop? A nested loop is a loop inside another loop. The inner loop executes completely for each iteration of the outer loop. This is essential for processing multidimensional data like 2D arrays (matrices), tables, or grids.

Simple nested loop example:

for (uint8_t outer = 0; outer < 3; outer++) {
    for (uint8_t inner = 0; inner < 2; inner++) {
        Serial.print(outer);
        Serial.print(",");
        Serial.println(inner);
    }
}
// Output:
// 0,0
// 0,1
// 1,0
// 1,1
// 2,0
// 2,1

// Total executions: 3 * 2 = 6 times

Important concept: If outer loop runs N times and inner loop runs M times, the inner body executes N × M times total.

Practical Embedded Example: Processing 2D Arrays

Scenario: You have a sensor grid (like a thermal camera) that reads temperature at different positions. Nested loops allow you to iterate through all rows and columns.

uint16_t temperature_grid[3][4] = {
    {200, 210, 215, 205},
    {220, 225, 230, 222},
    {215, 220, 225, 218}
};

uint16_t total_temp = 0;
for (uint8_t row = 0; row < 3; row++) {
    for (uint8_t col = 0; col < 4; col++) {
        uint16_t temp = temperature_grid[row][col];
        Serial.print("(");
        Serial.print(row);
        Serial.print(",");
        Serial.print(col);
        Serial.print("): ");
        Serial.println(temp);
        total_temp += temp;
    }
}

uint8_t average = total_temp / 12;
Serial.print("Average temperature: ");
Serial.println(average);

Image Processing Example

Real embedded use case: Processing a camera image to detect bright spots. This example iterates through every pixel in a 100x100 image and counts how many are brighter than a threshold.

uint8_t image[100][100];

uint16_t bright_pixel_count = 0;

for (uint8_t y = 0; y < 100; y++) {
    for (uint8_t x = 0; x < 100; x++) {
        uint8_t pixel = image[y][x];
        if (pixel > 200) {
            bright_pixel_count++;
        }
    }
}

Serial.print("Found ");
Serial.print(bright_pixel_count);
Serial.println(" bright pixels");

Performance Consideration: Nested Loop Overhead

Nested loops can be slow! The inner loop executes many times. If you do expensive calculations inside nested loops, performance suffers. Move calculations outside the loop if possible.

for (uint8_t row = 0; row < 100; row++) {
    for (uint8_t col = 0; col < 100; col++) {
        uint16_t converted = temperature_grid[row][col] * 9 / 5 + 32;
        process_temp(converted);
    }
}

Triple Nested Loops (3D Data)

Rarely needed but possible: Processing 3D data (like a cube of sensor values).

uint8_t sensor_cube[5][5][5];

for (uint8_t x = 0; x < 5; x++) {
    for (uint8_t y = 0; y < 5; y++) {
        for (uint8_t z = 0; z < 5; z++) {
            uint8_t value = sensor_cube[x][y][z];
        }
    }
}
⚠️ Performance Warning: Avoid Deep Nesting

Triple or quadruple nested loops are very slow. If you find yourself writing 4+ nested loops, it's often a sign that your data structure or algorithm needs rethinking. On microcontrollers with limited speed, deeply nested loops can cause your program to be too slow to respond to real-time events.

break and continue: Controlling Loop Flow

Sometimes you need to control the loop in special ways: Exit early, or skip to the next iteration. C++ provides two statements for this.

break: Exit Loop Early

The break statement immediately exits the current loop. Execution jumps to the first statement after the loop. This is useful when you find what you're looking for and don't need to continue searching.

uint8_t target = 67;
uint8_t found_index = 255;

for (uint8_t i = 0; i < 100; i++) {
    if (array[i] == target) {
        found_index = i;
        break;
    }
}

Serial.print("Found at: ");
Serial.println(found_index);

Another example: Communication timeout

uint8_t byte_received;
uint16_t timeout_count = 0;
const uint16_t MAX_TIMEOUT = 10000;

while (1) {
    if (serial_data_available()) {
        byte_received = serial_read();
        process_byte(byte_received);
        timeout_count = 0;
    }
    
    timeout_count++;
    if (timeout_count > MAX_TIMEOUT) {
        Serial.println("Communication timeout!");
        break;
    }
}

Common pattern: Reading until end marker

uint8_t buffer[256];
uint8_t bytes_received = 0;

for (uint8_t i = 0; i < 256; i++) {
    uint8_t byte = serial_read();
    
    if (byte == 0xFF) {
        bytes_received = i;
        break;
    }
    
    buffer[i] = byte;
}

process_message(buffer, bytes_received);

break in Nested Loops

Important: break only exits the IMMEDIATE loop, not all loops. If you need to exit from nested loops, use a flag variable.

for (uint8_t row = 0; row < 10; row++) {
    for (uint8_t col = 0; col < 10; col++) {
        if (array[row][col] == target) {
            Serial.println("Found!");
            break;
        }
    }
}

uint8_t found = 0;
for (uint8_t row = 0; row < 10; row++) {
    for (uint8_t col = 0; col < 10; col++) {
        if (array[row][col] == target) {
            found = 1;
            break;
        }
    }
    if (found) break;
}

continue: Skip to Next Iteration

The continue statement skips the rest of the current iteration and jumps to the next one. Useful for skipping invalid data or unwanted conditions. When continue is executed, the loop condition is checked immediately and the next iteration begins.

for (uint8_t i = 0; i < 100; i++) {
    uint16_t reading = sensor_array[i];
    
    if (reading < 10) {
        continue;
    }
    
    process_reading(reading);
    Serial.println(reading);
}

Another example: Skip certain hours for processing. When working with time-based data, you might want to skip certain time ranges.

for (uint8_t hour = 0; hour < 24; hour++) {
    if (hour >= 0 && hour < 6) {
        continue;
    }
    
    uint8_t temp = get_temperature_for_hour(hour);
    Serial.println(temp);
}

Pattern: Skipping Invalid Data

Very common in embedded: filter out bad sensor readings. Skip values that indicate errors or are out of range.

uint16_t valid_values[10];
uint8_t valid_count = 0;

for (uint8_t i = 0; i < total_readings; i++) {
    uint16_t value = get_sensor_reading(i);
    
    if (value == 0) {
        continue;
    }
    
    if (value > 4000) {
        continue;
    }
    
    valid_values[valid_count] = value;
    valid_count++;
}

Serial.print("Got ");
Serial.print(valid_count);
Serial.println(" valid readings");

Difference Between break and continue

uint8_t values[] = {5, 0, 15, 0, 25, 0, 35};

Serial.println("Using continue:");
for (uint8_t i = 0; i < 7; i++) {
    if (values[i] == 0) continue;
    Serial.println(values[i]);
}

Serial.println("Using break:");
for (uint8_t i = 0; i < 7; i++) {
    if (values[i] == 0) break;
    Serial.println(values[i]);
}

When to use break: You want to stop the loop entirely—you found what you're looking for.

When to use continue: You want to skip the current iteration but keep looping—you want to skip bad data but process more.

Infinite Loops (Embedded Staple!)

Embedded systems often run forever. Many use infinite loops as their main control structure.

void main() {
    init_hardware();
    init_tasks();
    
    // Infinite main loop
    while (1) {
        read_sensors();
        process_data();
        update_display();
        
        // Or with for loop
        // for (;;) { ... }
    }
}

// With clean shutdown
volatile uint8_t shutdown_requested = 0;

void main() {
    init_hardware();
    
    while (!shutdown_requested) {
        normal_operation();
    }
    
    cleanup_and_exit();
}

// From interrupt handler
void power_button_interrupt() {
    shutdown_requested = 1;
}

Why not infinite loops with RTOS?

When using a Real-Time Operating System (RTOS), each task should not have its own infinite loop. Instead, tasks should be designed to return, allowing the scheduler to run other tasks. The scheduler itself contains the infinite loop.

void sensor_task(void *param) {
    while (!shutdown_requested) {
        read_sensor();
        process_data();
        task_delay(100);
    }
}

void communication_task(void *param) {
    while (!shutdown_requested) {
        if (data_available()) {
            send_data();
        }
        task_delay(10);
    }
}

void main() {
    create_task(sensor_task);
    create_task(communication_task);
    start_scheduler();
}

Loop Optimization Techniques

In resource-constrained embedded systems, loop efficiency matters. Avoid expensive calculations in loop conditions and keep loop bodies as simple as possible. Countdown loops (comparing to 0) are sometimes faster than counting upward, and pre-calculating values outside loops avoids repeated work.

uint8_t my_array[100];
for (uint8_t i = 0; i < 100; i++) {
    process(my_array[i]);
}

for (uint16_t i = 0; i < strlen(buffer); i++) {
}

uint16_t len = strlen(buffer);
for (uint16_t i = 0; i < len; i++) {
}

for (uint8_t i = 100; i > 0; i--) {
}

Common Loop Mistakes

  • Off-by-one error: Loop runs one too many or too few times
  • Infinite loops: Condition never becomes false (forgot increment)
  • Modifying loop variable: Changes loop behavior unexpectedly
  • break/continue confusion: Only affect innermost loop
  • Unintended loop scope: Variable declared inside loops
uint8_t array[10];
for (uint8_t i = 0; i < 10; i++) {
    array[i] = i;
}

for (uint8_t i = 0; i < 10; i++) {
    if (i == 5) {
        i = 0;
    }
}

for (uint8_t i = 0; i < 10; i++) {
    for (uint8_t j = 0; j < 10; j++) {
        if (error) break;
    }
}

When to Use Which Loop

  • for: When iteration count is known or array has fixed size
  • while: When condition unknown upfront (reading input, waiting for event)
  • do-while: When body must execute at least once
  • break: When you need to exit loop early (find match, error condition)
  • continue: When you need to skip some iterations (filter data)

Summary

  • for loops: iteration count known, processing arrays
  • while loops: condition-based, unknown iteration count
  • do-while loops: when body must run at least once
  • Infinite loops common in embedded: while (1) or for (;;)
  • Use break to exit early, continue to skip iterations
  • Watch for off-by-one errors and infinite loops
  • Optimize loop conditions—avoid expensive calculations per iteration

Mini-Projects to Strengthen Your Skills

Project 1: Simple LED Blinking Pattern

Objective: Create a repeating blink pattern using a for loop.

What you'll learn: How loops let you repeat code multiple times without writing it over and over.

Step by step:

  1. Create variables: #define LED_PIN 13, a counter i, and blink_count = 5
  2. Write a for loop that counts from 0 to blink_count
  3. Inside the loop: digitalWrite(LED_PIN, HIGH), delay(500), digitalWrite(LED_PIN, LOW), delay(500)
#define LED_PIN 13

uint8_t blink_count = 5;

for (uint8_t i = 0; i < blink_count; i++) {
    digitalWrite(LED_PIN, HIGH);
    delay(500);
    digitalWrite(LED_PIN, LOW);
    delay(500);
}

Why this works: Without a loop, you'd write the same code 5 times! Loops handle repetition automatically. The variable i counts up, and the loop checks if i is still less than blink_count.

Project 2: Counting Up with Break

Objective: Count from 0 upward and stop early if a button is pressed using break.

What you'll learn: How break lets you exit a loop before it normally ends.

Step by step:

  1. Create variables: #define BUTTON_PIN 2, button_pressed = 0
  2. Use a while loop that counts: while (count < 100)
  3. Inside: increment count, check if button_pressed becomes 1
  4. If button is pressed, use break to immediately exit the loop
#define BUTTON_PIN 2

uint8_t count = 0;
uint8_t button_pressed = 0;

while (count < 100) {
    count++;
    
    // Check if button pressed
    if (digitalRead(BUTTON_PIN) == HIGH) {
        button_pressed = 1;
        break;  // Exit loop immediately
    }
    
    delay(100);
}

// If count reaches 100, loop ends normally
// If button pressed, break exits early and count < 100

Why this works: Loops normally run until their condition is false. Break provides an emergency exit—useful when something urgent happens (like a button press) that requires immediate action.

Project 3: Skipping Invalid Readings with Continue

Objective: Read 10 values but skip (continue) when reading is 0 (invalid).

What you'll learn: How continue skips to the next iteration without running the rest of the loop body.

Step by step:

  1. Create variables: #define SENSOR_PIN A0, valid_count = 0
  2. Use a for loop: for (uint8_t i = 0; i < 10; i++)
  3. Read sensor value: reading = analogRead(SENSOR_PIN)
  4. If reading == 0, use continue to skip the rest and go to next iteration
  5. Otherwise, increment valid_count and process the reading
#define SENSOR_PIN A0

uint8_t valid_count = 0;

for (uint8_t i = 0; i < 10; i++) {
    uint16_t reading = analogRead(SENSOR_PIN);
    
    // If reading is 0, it's invalid - skip to next iteration
    if (reading == 0) {
        continue;  // Jump to next i++
    }
    
    // Only reaches here for valid readings
    valid_count++;
    
    // Process the valid reading
    uint8_t scaled_value = (reading / 4);  // Convert 0-1023 to 0-255
    
    delay(100);
}

// After loop: valid_count has the number of non-zero readings
// Readings of 0 were skipped (not counted in valid_count)

Why this works: Continue skips the rest of the current iteration and jumps to the next one. It's perfect for filtering out bad data—instead of nesting everything deeply, you reject bad data early and keep going.
