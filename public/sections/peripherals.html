Peripherals & Communication Protocols

Peripherals are the interface between your microcontroller and the external world. This comprehensive section covers essential communication protocols (GPIO, UART, SPI, I2C) and sensor integration. Master these protocols to build interactive embedded systems that can sense and control their environment.

Learning Path: Begin with GPIO for basic I/O control, then learn synchronous (SPI) and asynchronous (UART) serial communication, followed by advanced protocols like I2C for multi-device communication.

Core Protocols & Interfaces

Learn essential communication protocols and I/O techniques:

1. GPIO (General Purpose I/O)

Digital input and output control. Learn pin configuration, debouncing, and implementing basic I/O patterns for LEDs, buttons, and switches.

Key Topics →
  • Pin configuration (Input, Output, Modes)
  • Push-pull and open-drain outputs
  • Pull-up and pull-down resistors
  • Debouncing techniques
  • Interrupt-driven GPIO

2. UART Serial Communication

Asynchronous serial communication for data transmission and debugging. Learn baud rates, framing, error detection, and multi-device communication.

Key Topics →
  • UART frame format and timing
  • Baud rate configuration
  • Start, stop, and parity bits
  • Interrupt-driven reception/transmission
  • Circular buffers for data handling
  • RS-232 and TTL levels

3. SPI (Serial Peripheral Interface)

Synchronous serial communication for fast, full-duplex data transfer. Master chip select, clock polarity, and multi-slave configurations.

Key Topics →
  • SPI bus configuration (MOSI, MISO, SCK, CS)
  • Clock polarity and phase (CPOL/CPHA)
  • Chip select handling
  • Master and slave operation
  • DMA and interrupt-driven transfers
  • Multi-slave selection strategies

4. I2C (Inter-Integrated Circuit)

Multi-master, multi-slave synchronous communication using only 2 wires. Ideal for sensor networks and modular systems.

Key Topics →
  • I2C bus protocol (START, STOP, ACK/NACK)
  • 7-bit and 10-bit addressing
  • Clock stretching and bus arbitration
  • I2C speed modes (Standard, Fast, Fast+)
  • Pull-up resistor sizing
  • Multi-master arbitration

5. PWM (Pulse Width Modulation)

Analog signal simulation through digital pulses. Essential for motor control, LED brightness, and servo control.

Key Topics →
  • Duty cycle and frequency control
  • Timer configuration for PWM
  • Motor speed control
  • LED brightness adjustment
  • Servo position control
  • Dead-time insertion (H-bridge control)

6. ADC (Analog-to-Digital Conversion)

Convert analog signals to digital values. Learn sampling rates, resolution, and filtering for sensor measurements.

Key Topics →
  • ADC resolution and quantization
  • Sampling rate and Nyquist theorem
  • Single-ended and differential inputs
  • ADC calibration and accuracy
  • DMA-based continuous conversion
  • Sensor signal conditioning

7. Timers & Counters

Time measurement, delay generation, event counting, and capture modes. Critical for real-time applications.

Key Topics →
  • Timer modes (Compare, Capture, PWM)
  • Prescaler configuration
  • Input capture for pulse measurement
  • Edge detection and frequency counting
  • Hardware timing accuracy
  • Watchdog timers

8. Sensor Integration

Interface common sensors: temperature, pressure, acceleration, proximity. Learn calibration and data interpretation.

Key Topics →
  • Temperature sensor types and interfaces
  • Pressure and IMU sensors (I2C/SPI)
  • Proximity and distance measurement
  • Sensor calibration techniques
  • Noise filtering and averaging
  • Sensor fusion basics

9. CAN Bus (Advanced)

Vehicle and industrial automation communication. Learn message framing, arbitration, and multi-node networks.

Key Topics →
  • CAN message format and identifiers
  • Arbitration and priority
  • Error detection and handling
  • CAN network design
  • Protocol stack (DBC, CANopen)
  • Transceiver interface

Protocol Comparison & Selection

Protocol Speed Nodes Wires Best For
GPIO Unlimited 2 (1:1) 1 Simple I/O, buttons, LEDs
UART Up to 115.2 kbps 2 (1:1) 2-3 Debugging, serial protocols
SPI Up to 50 Mbps Many (M:S) 4+ (1 per slave) Fast parallel devices, SD cards
I2C Up to 3.4 Mbps Many (M:M) 2 Sensors, EEPROMs, scalable
CAN Up to 1 Mbps Many (broadcast) 2 Automotive, industrial networks

Common Sensor Integration Examples

  • Temperature Sensors: DS18B20 (1-Wire), LM75 (I2C), thermistors with ADC
  • Motion & Orientation: MPU-6050 (I2C accelerometer/gyro), HMC5883L (magnetometer)
  • Environmental Sensors: BME280 (temperature/humidity/pressure via I2C), DHT22
  • Distance & Proximity: HC-SR04 (ultrasonic), VL53L0X (Time-of-Flight)
  • Light Sensors: BH1750 (I2C ambient light), LDR with ADC
  • Storage & Identification: 24C256 EEPROM (I2C), RFID readers (UART/SPI)
