Functions in C++

What's a function? A function is a reusable recipe—like a cooking recipe that tells you step-by-step how to make something. Instead of writing the same steps over and over, you write them once and then just say "do this recipe!"

Example recipe: "To bake a cookie, do: mix flour, add butter, add sugar, add eggs, bake for 10 minutes." Instead of writing those steps 50 times for 50 batches, you just say "bake a cookie" and the computer follows the recipe.

Why use functions?

  • Less repetition: Write code once, use it many times
  • Easier to understand: Complex code is broken into smaller, named chunks
  • Easier to fix mistakes: Fix it in one place, fixes it everywhere
  • Team work: Different people can write different functions

Embedded examples:

  • init_uart() - Sets up the serial port (written once, used forever)
  • read_temperature() - Gets the current temperature (called many times)
  • blink_led() - Turns LED on, waits, turns off (simple and reusable)

Function Declaration and Definition

Two steps to using a function:

  • Declaration (promise): "I promise there's a function called say_hello that doesn't take any inputs and returns nothing"
  • Definition (delivery): "Here's the actual code that say_hello does"

Why two steps? The compiler needs to know a function exists before you try to use it. It's like telling your friend "I'm going to call you at 3pm" (declaration) before actually calling them (definition/execution).

A function must be declared before it's used, then defined with implementation.

void say_hello();
uint8_t add_numbers(uint8_t a, uint8_t b);

void say_hello() {
    uart_print("Hello!");
}

uint8_t add_numbers(uint8_t a, uint8_t b) {
    return a + b;
}

int main() {
    say_hello();
    uint8_t result = add_numbers(5, 3);
}

Parameters and Arguments

Parameters are variables in the function definition. Arguments are the actual values passed.

// Define function with parameters
void set_brightness(uint8_t level) {
    PWM_VALUE = level;
}

// Call function with arguments
set_brightness(100);  // 100 is the argument

// Multiple parameters
void configure_uart(uint32_t baud, uint8_t parity) {
    UART_BAUD = baud;
    UART_PARITY = parity;
}

// Call with values
configure_uart(9600, PARITY_NONE);

Pass by Value

By default, parameters are passed by value (a copy is made). Changes inside the function don't affect the original variable.

void increment(uint8_t value) {
    value++;
}

uint8_t count = 5;
increment(count);  // Pass 5 (copy)
// count is still 5 - function modified the copy, not original

// To modify the original, use pointers (covered next)
void increment_ptr(uint8_t *value) {
    (*value)++;  // Modifies original
}

count = 5;
increment_ptr(&count);  // Pass address
// count is now 6 - function modified original

// Parameter with default value
void delay_ms(uint16_t ms = 100) {

// Call with argument
delay_ms(500);     // Wait 500ms

void uart_send_string(const char *str, uint8_t newline = 1) {

Functions return exactly one value (can be 0 or a structure containing multiple values).

    uint16_t raw = ADC_READ();

    return celsius;

uint16_t temp = read_temperature();

// Error codes
uint8_t send_packet(uint8_t *data, uint8_t length) {

        return ERROR_NULL_POINTER;

uint8_t result = send_packet(my_data, 32);

Variable Scope in Functions

Variables declared in a function are local to that function.

uint8_t global_count = 0;  // Global scope

void my_function() {
    uint8_t local_count = 5;  // Local scope
    
    global_count++;           // Can access global
    local_count++;            // Can access local
}

int main() {
    global_count = 10;        // Can access global
    // local_count = 15;       // ERROR - not in scope here
    
    my_function();            // local_count created and destroyed
    // local_count doesn't exist anymore
}

Static Function Variables

A static variable in a function persists between calls.

void increment_counter() {
    static uint16_t counter = 0;  // Initialized once
    counter++;
    uart_print_int(counter);
}

int main() {
    increment_counter();  // Prints 1
    increment_counter();  // Prints 2
    increment_counter();  // Prints 3
}

// Useful for state machines and counters
uint8_t get_next_id() {
    static uint8_t id = 0;
    id++;
    return id;
}

get_next_id();  // Returns 1
get_next_id();  // Returns 2
get_next_id();  // Returns 3

Function Pointers and Callbacks

Pointers to functions enable powerful patterns like callbacks in embedded systems.

// Define a function pointer type
typedef void (*CALLBACK_T)(uint8_t);

// Function that matches the callback signature
void on_button_press(uint8_t pin) {
    uart_print("Button pressed on pin ");
    uart_print_int(pin);
}

// Register callback
CALLBACK_T button_callback = on_button_press;

// Call it
button_callback(5);  // Same as on_button_press(5)

// Interrupt handler pattern
CALLBACK_T uart_rx_handler = NULL;

void uart_interrupt() {
    uint8_t byte = UART_READ_DATA;
    if (uart_rx_handler != NULL) {
        uart_rx_handler(byte);  // Call registered handler
    }
}

void on_uart_receive(uint8_t byte) {
    buffer_put(byte);
}

void init() {
    uart_rx_handler = on_uart_receive;  // Register
}

Callback Use Cases

// Event dispatcher
typedef void (*EVENT_HANDLER)(uint16_t);

struct EventHandler {
    uint16_t event_type;
    EVENT_HANDLER handler;
};

// Register multiple handlers
EventHandler handlers[10];
uint8_t handler_count = 0;

void register_handler(uint16_t event, EVENT_HANDLER handler) {
    handlers[handler_count].event_type = event;
    handlers[handler_count].handler = handler;
    handler_count++;
}

void dispatch_event(uint16_t event) {
    for (uint8_t i = 0; i < handler_count; i++) {
        if (handlers[i].event_type == event) {
            handlers[i].handler(event);
        }
    }
}

void on_temperature_threshold(uint16_t temp) {
    activate_cooling();
}

void on_motion_detected(uint16_t sensor_id) {
    log_motion(sensor_id);
}

void init() {
    register_handler(EVENT_TEMP_HIGH, on_temperature_threshold);
    register_handler(EVENT_MOTION, on_motion_detected);
}

Inline Functions

Tell compiler to inline (expand) function at call sites for speed.

// Small functions that get called often
static inline uint8_t is_button_pressed() {
    return (GPIO_READ(BUTTON_PIN) == 0);
}

static inline void set_led(uint8_t state) {
    GPIO_WRITE(LED_PIN, state);
}

// Usage - compiled as inline code, not function call
if (is_button_pressed()) {
    set_led(1);
}

// Compiler generates:
// if ((GPIO_READ(BUTTON_PIN) == 0)) {
//     GPIO_WRITE(LED_PIN, 1);
// }

// Benefits:
// - No function call overhead (push/pop stack)
// - Compiler can optimize further
// - Faster for critical code paths

// Drawbacks:
// - Code size increases (function body copied)
// - Not suitable for large functions

Recursion

Functions can call themselves. Use with care in embedded systems!

// Calculate factorial
uint32_t factorial(uint8_t n) {
    if (n <= 1) {
        return 1;  // Base case
    }
    return n * factorial(n - 1);  // Recursive call
}

factorial(5);  // 5 * 4 * 3 * 2 * 1 = 120

// Binary search (useful for embedded)
uint8_t binary_search(uint8_t *arr, uint8_t size, uint8_t target) {
    return binary_search_helper(arr, 0, size - 1, target);
}

uint8_t binary_search_helper(uint8_t *arr, uint8_t left, uint8_t right, uint8_t target) {
    if (left > right) {
        return NOT_FOUND;
    }
    
    uint8_t mid = left + (right - left) / 2;
    if (arr[mid] == target) {
        return mid;
    } else if (arr[mid] < target) {
        return binary_search_helper(arr, mid + 1, right, target);
    } else {
        return binary_search_helper(arr, left, mid - 1, target);
    }
}

// WARNING: Recursion uses stack memory
// Each function call adds to the stack
// With limited RAM in embedded systems, stack can overflow!
// Avoid deep recursion

const and Function Parameters

Use const to promise not to modify data.

// Parameter that won't be modified
void play_tone(const uint16_t *frequency) {
    // Compiler prevents: *frequency = 1000;
}

// Pointer to constant data
void send_data(const uint8_t *buffer, uint8_t length) {
    for (uint8_t i = 0; i < length; i++) {
        uart_tx(buffer[i]);  // Can read
        // buffer[i] = 0;  // ERROR - can't modify
    }
}

// Benefits:
// - Intent is clear: this function doesn't modify your data
// - Compiler catches mistakes
// - Enables more optimizations

Common Function Patterns

Getters and Setters

// Encapsulate internal state
typedef struct {
    uint8_t pwm_value;
} Motor;

Motor motor = {0};

// Getter
uint8_t motor_get_speed(Motor *m) {
    return m->pwm_value;
}

// Setter with validation
void motor_set_speed(Motor *m, uint8_t speed) {
    if (speed > 255) {
        speed = 255;  // Clamp
    }
    m->pwm_value = speed;
}

Initialization Functions

// Initialize module
void uart_init(uint32_t baud, uint8_t parity) {
    UART_BAUD_RATE = baud;
    UART_PARITY = parity;
    UART_CONTROL = UART_ENABLE;
}

void i2c_init(uint16_t clock_speed) {
    I2C_CLOCK_SPEED = clock_speed;
    I2C_CONTROL = I2C_ENABLE;
}

void app_init() {
    uart_init(9600, PARITY_NONE);
    i2c_init(400000);  // 400 kHz
}

Polling Functions

// Return current state
uint8_t is_data_available() {
    return (UART_STATUS & UART_RX_FULL) != 0;
}

// Called repeatedly
while (1) {
    if (is_data_available()) {
        uint8_t byte = uart_read();
        process_byte(byte);
    }
}

Common Function Mistakes

  • Not declaring function first: Can't call function before it's declared
  • Wrong parameter types: Passing wrong type leads to incorrect behavior
  • Forgetting return value: Function returns undefined value if no return statement
  • Stack overflow: Too many function calls or deep recursion exhausts stack
  • Not checking for NULL: Using pointer parameter without checking if NULL
// WRONG - Function called before declared
main() {
    my_function();  // ERROR - not declared yet
}

void my_function() {
    // ...
}

// RIGHT - Use forward declaration
void my_function();  // Declare first

main() {
    my_function();  // OK
}

void my_function() {
    // ...
}

// WRONG - No return value
uint8_t get_temperature() {
    uint16_t raw = ADC_READ();
    // Forgot return statement!
}

// RIGHT
uint8_t get_temperature() {
    uint16_t raw = ADC_READ();
    return raw;
}

// WRONG - Unchecked pointer
void process_data(uint8_t *data) {
    uint8_t value = data[0];  // Crashes if data is NULL
}

// RIGHT
void process_data(uint8_t *data) {
    if (data == NULL) {
        return ERROR_NULL;
    }
    uint8_t value = data[0];  // Safe
}

Function Stack Usage and Embedded Constraints

Every function call uses stack memory. With limited RAM (2KB-256KB in many microcontrollers), you must be aware of stack usage.

// Each call uses stack space
void function_a() {
    uint8_t buffer_a[100];  // 100 bytes on stack
}

void function_b() {
    uint16_t data[50];      // 100 bytes on stack
    function_a();           // Nested call adds more stack
}

void function_c() {
    uint32_t value;         // 4 bytes on stack
    function_b();           // Calls function_b which calls function_a
                            // Stack now has all three functions' locals
}

// Deep nesting example:
void level_1() { uint8_t a[100]; level_2(); }
void level_2() { uint8_t b[100]; level_3(); }
void level_3() { uint8_t c[100]; level_4(); }
void level_4() { uint8_t d[100]; }

// level_4 stack: 400 bytes just from local arrays
// Add function return addresses and you could overflow!

// SOLUTION: Use static buffers instead
static uint8_t buffer_pool[256];  // Single shared buffer

void process_data_safe() {
    // Reuse buffer_pool instead of allocating on stack
}

Stack overflow signs:

  • Program crashes unexpectedly
  • Variables change value mysteriously
  • Hard faults or watchdog resets

Solutions:

  • Limit nesting depth (function A calls B calls C, not deeper)
  • Use static/global buffers for large data
  • Reduce local variable sizes
  • Profile stack usage with compiler tools

Function Attributes for Embedded Systems

GCC allows function attributes to control code generation on ARM and AVR microcontrollers.

// Weak function: Can be overridden elsewhere
void uart_init() __attribute__((weak));
void uart_init() {
    // Default implementation
    UART_BAUD = 9600;
}
// If another file defines uart_init without weak, that version is used


// Interrupt handler (ARM Cortex-M)
void UART0_IRQHandler() __attribute__((interrupt));
void UART0_IRQHandler() {
    // Compiler adds function prologue/epilogue for interrupt
    uint8_t byte = UART_READ();
    buffer_put(byte);
}


// No-return function (exits, never returns)
void panic() __attribute__((noreturn));
void panic() {
    uart_print("PANIC!");
    while (1);  // Infinite loop
}
// Compiler knows not to generate return code


// Always inline (force inlining)
static void __attribute__((always_inline)) get_pin_fast(uint8_t pin) {
    return PIN_READ(pin);
}


// Packed struct for memory-mapped hardware registers
typedef struct __attribute__((packed)) {
    uint8_t control;     // Offset 0
    uint8_t status;      // Offset 1
    uint16_t data;       // Offset 2
} HardwareReg;

// Without packed, compiler might add padding

Advanced Pattern: State Machines

Functions and callbacks enable state machine patterns common in embedded systems.

// State machine using function pointers
typedef enum {
    STATE_IDLE,
    STATE_READING,
    STATE_PROCESSING,
    STATE_SENDING
} SystemState;

typedef void (*StateFunc)();

SystemState current_state = STATE_IDLE;
StateFunc state_functions[4];

void state_idle() {
    if (sensor_data_available()) {
        current_state = STATE_READING;
    }
}

void state_reading() {
    raw_value = adc_read();
    current_state = STATE_PROCESSING;
}

void state_processing() {
    processed_value = apply_filter(raw_value);
    current_state = STATE_SENDING;
}

void state_sending() {
    uart_send_value(processed_value);
    current_state = STATE_IDLE;
}

void system_init() {
    state_functions[STATE_IDLE] = state_idle;
    state_functions[STATE_READING] = state_reading;
    state_functions[STATE_PROCESSING] = state_processing;
    state_functions[STATE_SENDING] = state_sending;
}

void main_loop() {
    while (1) {
        state_functions[current_state]();  // Call appropriate state handler
        delay_ms(10);
    }
}

// Benefits:
// - Clear state transitions
// - Each state is a separate function
// - Easy to add new states
// - Scales to complex embedded applications

Real-World Embedded Function Examples

Example 1: I2C Communication

// Read multiple bytes from I2C device
uint8_t i2c_read_register(uint8_t address, uint8_t reg, uint8_t *data, uint8_t length) {
    if (data == NULL) return ERROR_NULL;
    if (length == 0) return ERROR_INVALID_LENGTH;
    
    // Start condition
    i2c_start();
    
    // Send device address + read bit
    i2c_send_byte((address << 1) | 1);
    
    if (!i2c_wait_ack(TIMEOUT_MS)) {
        i2c_stop();
        return ERROR_NO_ACK;
    }
    
    // Read data bytes
    for (uint8_t i = 0; i < length; i++) {
        data[i] = i2c_read_byte();
        
        // Send ACK for all but last byte
        if (i < length - 1) {
            i2c_send_ack();
        } else {
            i2c_send_nack();
        }
    }
    
    i2c_stop();
    return SUCCESS;
}

Example 2: ADC Averaging

// Read ADC multiple times and return average
uint16_t adc_read_average(uint8_t channel, uint8_t samples) {
    uint32_t sum = 0;
    
    // Avoid reading too many samples (stack usage)
    if (samples > 100) samples = 100;
    
    for (uint8_t i = 0; i < samples; i++) {
        sum += adc_read(channel);
        delay_us(100);  // Small delay between reads
    }
    
    return (uint16_t)(sum / samples);
}

// Usage:
uint16_t stable_temp = adc_read_average(ADC_TEMP_CHANNEL, 10);

Example 3: Safe UART Send

// Send data with timeout (prevents hanging)
uint8_t uart_send_with_timeout(const uint8_t *data, uint16_t length, uint16_t timeout_ms) {
    if (data == NULL) return ERROR_NULL;
    
    for (uint16_t i = 0; i < length; i++) {
        uint32_t start = system_ticks();
        
        // Wait until TX ready or timeout
        while ((uart_get_status() & UART_TX_READY) == 0) {
            if ((system_ticks() - start) > timeout_ms) {
                return ERROR_TIMEOUT;
            }
        }
        
        uart_write_byte(data[i]);
    }
    
    return SUCCESS;
}

// Usage:
uint8_t result = uart_send_with_timeout(packet, packet_length, 100);
if (result != SUCCESS) {
    log_error("UART send timeout");
}

Example 4: Debounced Button Read

// Read button multiple times to verify press (debounce)
uint8_t button_is_pressed_debounced(uint8_t pin, uint8_t checks) {
    uint8_t pressed_count = 0;
    
    for (uint8_t i = 0; i < checks; i++) {
        if (gpio_read(pin) == 0) {  // Button pulls to GND when pressed
            pressed_count++;
        }
        delay_ms(10);  // Small delay between checks
    }
    
    // Button is considered pressed if ALL checks read low
    return (pressed_count == checks);
}

// Usage:
if (button_is_pressed_debounced(BUTTON_PIN, 5)) {
    handle_button_press();
}

Function Design Best Practices

Single Responsibility: Each function does ONE thing.

// BAD - does too much
void read_and_process_and_send_data() {
    uint16_t raw = adc_read();
    uint16_t processed = apply_filter(raw);
    uart_send_value(processed);
}

// GOOD - separated concerns
uint16_t read_sensor() { return adc_read(); }
uint16_t process_value(uint16_t raw) { return apply_filter(raw); }
void send_value(uint16_t value) { uart_send_value(value); }

// Main loop clearly shows the steps
while (1) {
    uint16_t raw = read_sensor();
    uint16_t processed = process_value(raw);
    send_value(processed);
}

Defensive Programming: Validate inputs.

// Check for NULL, invalid ranges, etc.
uint8_t motor_set_speed(Motor *m, uint8_t speed) {
    if (m == NULL) return ERROR_NULL;
    if (speed > MAX_SPEED) return ERROR_OUT_OF_RANGE;
    
    m->speed = speed;
    return SUCCESS;
}

Consistent Naming: Names describe what function does.

// GOOD names
void enable_interrupt();
void read_temperature_sensor();
uint8_t get_battery_voltage();
void delay_milliseconds(uint16_t ms);

// BAD names
void do_stuff();           // What stuff?
void x();                  // Meaningless
void function123();        // What does it do?

Function Performance in Embedded Systems

Function call overhead:

// Every function call includes:
// 1. PUSH return address (2-4 bytes on stack)
// 2. PUSH saved registers (varies)
// 3. Allocate local variables
// 4. Function body
// 5. POP local variables
// 6. POP saved registers
// 7. RET (jump back to caller)

// For inner loops, this overhead matters!
// Compare:

// Version 1: Function call (overhead!)
unsigned long ticks = 0;
for (int i = 0; i < 1000; i++) {
    if (is_button_pressed()) {  // Function call, ~10 cycles overhead
        ticks++;
    }
}

// Version 2: Inline (no function call overhead)
unsigned long ticks = 0;
for (int i = 0; i < 1000; i++) {
    if ((GPIO_READ(BUTTON_PIN) == 0)) {  // Direct, ~2 cycles
        ticks++;
    }
}

// Solution: Use inline for frequently called small functions
static inline uint8_t is_button_pressed() {
    return (GPIO_READ(BUTTON_PIN) == 0);
}

Code size vs speed trade-offs:

// Small RAM? Use functions (saves code)
uint8_t add(uint8_t a, uint8_t b) { return a + b; }
uint8_t sub(uint8_t a, uint8_t b) { return a - b; }

// Fast loop needed? Inline small functions
static inline uint8_t add_fast(uint8_t a, uint8_t b) { return a + b; }

Project 1: Reusable Blink Helper

Objective: Create a function that blinks an LED a specified number of times.

What you'll learn: How functions eliminate code repetition and make programs easier to understand.

Step by step:

  1. Write a function: void blinkLED(uint8_t pin, uint8_t times)
  2. Inside: use digitalWrite and delay to blink the specified number of times
  3. Call it multiple times with different pins and counts
  4. Notice how the code becomes cleaner and more readable
// Function declaration (promise)
void blinkLED(uint8_t pin, uint8_t times);

// Function definition (implementation)
void blinkLED(uint8_t pin, uint8_t times) {
    for (uint8_t i = 0; i < times; i++) {
        digitalWrite(pin, HIGH);
        delay(200);
        digitalWrite(pin, LOW);
        delay(200);
    }
}

void loop() {
    // Now we can use the function easily
    blinkLED(13, 3);   // Red LED: 3 blinks
    delay(1000);
    
    blinkLED(12, 5);   // Green LED: 5 blinks
    delay(1000);
}

// Without functions, we'd copy-paste the loop code multiple times!

Why this works: Functions package code into a reusable recipe. Need to blink different pins? Just call the function with different parameters. Much cleaner than copying code!

Project 2: Temperature Converter Function

Objective: Create a function that converts raw sensor readings to temperature and returns the result.

What you'll learn: How functions can take input and return calculated results.

Step by step:

  1. Write: uint8_t convertRawToTemp(uint16_t raw_value)
  2. Inside: scale the raw value (0-1023) to temperature (0-100°C)
  3. Return the calculated temperature
  4. Call it with different raw values and store/print the results
// Function that returns a value
uint8_t convertRawToTemp(uint16_t raw_value) {
    // Scale 0-1023 to 0-100°C
    uint8_t temp = (raw_value / 1023) * 100;
    return temp;  // Send the result back to caller
}

void setup() {
    Serial.begin(9600);
}

void loop() {
    // Get a raw reading from sensor
    uint16_t raw = analogRead(A0);
    
    // Call function and store the result
    uint8_t temperature = convertRawToTemp(raw);
    
    // Use the result
    Serial.print("Temperature: ");
    Serial.print(temperature);
    Serial.println("°C");
    
    delay(1000);
}

// Notice: convertRawToTemp() does ONE thing—convert raw to temp
// Single Responsibility Principle!

Why this works: Functions with return values let you compute something and send the answer back. The caller can then use that result. This separation means the conversion logic is independent and reusable.

Project 3: Multi-Purpose Delay Helper

Objective: Create multiple versions of a function using function overloading or parameters with defaults.

What you'll learn: How functions can be flexible with optional parameters or different behaviors.

Step by step:

  1. Write: void wait(uint16_t milliseconds)
  2. Write: void waitSeconds(uint8_t seconds)
  3. The second calls the first internally: wait(seconds * 1000)
  4. Call both and see how they save you from remembering ms vs seconds
// Low-level delay in milliseconds
void wait(uint16_t milliseconds) {
    delay(milliseconds);
}

// High-level delay in seconds
void waitSeconds(uint8_t seconds) {
    wait(seconds * 1000);  // Convert to ms and call lower function
}

void loop() {
    // Wait 500ms (directly using milliseconds)
    wait(500);
    
    // Wait 2 seconds (easier to read)
    waitSeconds(2);
    
    // Wait 5 seconds without thinking about conversion
    waitSeconds(5);
}

// This layering pattern appears everywhere:
// - High-level: simple, human-readable
// - Low-level: does actual work
// High-level calls low-level internally

Why this works: By creating two functions at different levels of abstraction, users can call whichever makes sense. Someone who thinks in seconds calls waitSeconds(). Someone who needs precision calls wait(). Both work correctly!

Summary

  • Functions organize code into reusable blocks
  • Each function has one purpose (single responsibility)
  • Parameters passed by value; use pointers for modification
  • Functions return exactly one value (use structures for multiple)
  • Local variables only exist within function scope
  • Use inline for small, frequently-called functions
  • Use pointer parameters for output and modification
  • Check for NULL pointers and out-of-range values
  • Callbacks enable flexible event handling patterns
